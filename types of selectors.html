ВИДИ СЕЛЕКТОРІВ

  • A - E{ Опис правил стилю }Тут E ім'я довільного елемента. Слід розуміти, що хоча правила стилю можна застосувати до будь-якого елемента, результат буде помітний тільки для елементів, які безпосередньо відображаються в контейнері body.

  • #id -  <Ім'я ідентифікатора> { Опис правил стилю },Ідентифікатор визначає унікальне ім'я елемента, яке використовується для зміни його стилю і звернення до нього через скрипти. На відміну від класів ідентифікатори мають бути унікальними, іншими словами, зустрічатися в коді документа тільки один раз. Звернення до ідентифікатора відбувається за допомогою атрибута id, значенням якого виступає ім'я ідентифікатора (id="header").

  • .class -  декількох елементів веб-сторінки. Елементи при цьому можуть бути різними, але мати частково або повністю схожі елементи оформлення. У коді HTML ім'я класу визначається через атрибут class.

  • .class1.class2 -  <"e class="class1 class2 ...> { Опис правил стилю }, до будь-якого елемента одночасно можна додати кілька класів, перераховуючи їх в атрибуті class через пробіл. У цьому разі до елемента застосовується стиль, описаний у правилах для кожного класу.

  • * -  * { Опис правил стилю }, для позначення універсального селектора застосовується символ зірочки (*). У деяких випадках вказувати універсальний селектор не обов'язково. Так, наприклад, записи *.class і .class є ідентичними за своїм результатом.

  • A B -  E F { Опис правил стилю }, тут E це батьківський елемент, а F — дочірній елемент. Не обов'язково має бути два елементи, допускається довільний рівень вкладення. У цьому конструкція може записуватися так: div div ul li {...}.

  • A>B - E>F { Опис правил стилю }, стиль застосовується до елемента F, тільки коли він є дочірнім елементом E.

  • A + B -  E + F { Опис правил стилю }, сусідні називаються елементи веб-сторінки, коли вони йдуть безпосередньо один за одним у коді документа.

  • A ~ B -  E ~ F { Опис правил стилю }, для керування стилем споріднених елементів використовується символ тільди (~), його додають між двома селекторами E і F.

  • E + F {Опис правил стилю } -  Сусідні називаються елементи веб-сторінки, коли вони йдуть безпосередньо один за одним у коді документа.

  • [attr] -  ["<"атрибут">"] { Опис правил стилю }, стиль застосовується до тих елементів, усередині яких додано вказаний атрибут. Пробіл між ім'ям селектора і квадратними дужками не допускається.

  • [attr='value'] - Встановлює стиль для елемента в тому випадку, якщо задано певне значення специфічного атрибута.[атрибут="<значення>"] { Опис правил стилю }, E[атрибут="<значення>" ] { Опис правил стилю }, у першому випадку стиль застосовується до всіх тегів, які містять вказане значення атрибута. А в другому - тільки до певних селекторів.

  • [attr^='value'] -  Встановлює стиль для елемента в тому випадку, якщо значення атрибута починається із зазначеного тексту.[атрибут^="<значення>"] { Опис правил стилю },[атрибут^="<значення>"] { Опис правил стилю }, у першому випадку стиль застосовується до всіх елементів, у яких значення атрибута починаються із зазначеного тексту. А в другому - тільки до певних селекторів. Використання лапок не обов'язкове.

  • [attr$='value'] -  Встановлює стиль для елемента в тому випадку, якщо значення атрибута закінчується зазначеним текстом.[<атрибут>$="<значення>"] { Опис правил стилю }, E[<атрибут>$="<значення>"] { Опис правил стилю }, у першому випадку стиль застосовується до всіх елементів, у яких значення атрибута завершується заданим текстом. А в другому - тільки до певних селекторів.

  • [attr*='value'] -  Можливі варіанти, коли стиль слід застосувати до тега з певним атрибутом, коли частиною його значення є деякий текст. При цьому точно не відомо, в якому місці значення включено цей текст - на початку, в середині або наприкінці.

  • [attr~='value'] -  Деякі значення атрибутів можуть перераховуватися через пробіл, наприклад імена класів. Щоб задати стиль за наявності в списку необхідного значення, потрібно використовувати конструкцію ~=.

  • [attr|='value'] -  В іменах ідентифікаторів і класів дозволено використовувати символ дефіса (-), що дає змогу створювати значущі значення атрибутів id і class.Стиль застосовується до елементів, у яких атрибут точно дорівнює зазначеному значенню або починається із зазначеного значення, після якого йде дефіс.
